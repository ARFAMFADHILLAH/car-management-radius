1. Gambaran Umum Aplikasi

Nama Aplikasi (contoh):
Sistem Peminjaman Mobil Operasional
Perusahaan: PT Radius Allkindo Electric

Tujuan Aplikasi:

Mengelola permintaan penggunaan mobil operasional

Menghindari bentrok jadwal mobil

Mempermudah admin melakukan verifikasi & pencatatan

Menggantikan form manual / Google Form

2. Fitur yang Ada di Aplikasi
   A. Fitur untuk User (Karyawan)

Form Request Peminjaman Mobil

Isi data sesuai form yang kamu jelaskan

Cek Ketersediaan Mobil

Melihat jadwal mobil yang sudah terpakai (tabel kalender/list)

Riwayat Pengajuan

Melihat status pengajuan sendiri

Notifikasi Status

Status: Menunggu, Disetujui, Ditolak, Selesai

B. Fitur untuk Admin

Login Admin

Manajemen Data Mobil

Tambah / edit / hapus mobil

Manajemen Pengajuan

Lihat semua request

Setujui / tolak permintaan

Cek Bentrok Jadwal

Sistem otomatis mendeteksi jadwal tabrakan

Laporan Peminjaman

Filter berdasarkan tanggal, mobil, divisi

Export Data (Opsional)

PDF / Excel

C. Status Pengajuan

pending → menunggu persetujuan

approved → disetujui

rejected → ditolak

returned → mobil sudah kembali

3. Desain Database (Struktur Tabel)
   Tabel Utama:

users (pengguna)

divisions (divisi)

cars (mobil)

car_requests (pengajuan peminjaman)
